# SYSTEM ARCHITECTURE SPECIFICATION
# SYSTEM: Headless, Code-First, Instance-Isolated ERP Core

## 1. TOPOLOGY DIAGRAM
[ React/Next.js UI ] <---(Async REST / WebSockets)---> [ FastAPI Application ]
                                                               | (AsyncEngine)
                                                               v
                                                      [ PostgreSQL DB ]

## 2. DIRECTORY STRUCTURE & EXPLICIT REGISTRY
The backend heavily relies on a strict directory tree for automated feature mounting, enforcing module isolation and reducing circular dependencies.

/backend
├── app/
│   ├── main.py                  [BOOTSTRAP: Mounts Core + Initializes Workspaces]
│   ├── database.py              [ENGINE: AsyncSession pool configuration]
│   ├── core/                    [IMMUTABLE LAYER: Required for all deployments]
│   │   ├── auth/                (Models, Schemas, Routers for JWT/Users)
│   │   ├── directory/           (Models, Schemas, Routers for Customers/Sites)
│   │   ├── catalog/             (Models, Schemas, Routers for Base Items)
│   │   ├── events/              (Models, Schemas, Routers for WebSockets)
│   │   └── workspace/registry.py(Master registry for pluggable workspaces)
│   └── workspaces/              [PLUG-AND-PLAY LAYER]
│       ├── crm/                 (Sales Orders, Customers, Tasks)
│       ├── finance/             (Journal Entries, Ledger)
│       ├── hr/                  (Employees, Departments)
│       ├── inventory/           (Items, Warehouses, Stock Ledger)
│       └── operations/          (Manufacturing, Fulfillment)

## 3. DYNAMIC BOOTSTRAPPER LOGIC
- On `uvicorn` startup, `main.py` explicitly mounts all `/api/v1/core/*` routes.
- `main.py` calls `init_workspaces(app)`.
- `init_workspaces` iterates through `WORKSPACE_REGISTRY` (a dictionary mapping names to APIRouters) and mounts each to `/api/v1/workspaces/{subdirectory_name}`.
- Benefit: Prevents circular import failures, explicitly dictates module load order, and eliminates the unpredictable nature of `importlib` auto-scanning while retaining a pluggable pattern.

## 4. INTER-INSTANCE SYNC PIPELINE
- Inter-company communication is strictly API-based (Webhooks). 
- No shared databases. No shared Redis clusters.
- Pipeline: Node A executes a background worker task -> Formats a secure Pydantic JSON payload -> Transmits via HTTPS POST to Node B's ingestion endpoint -> Node B validates and writes to its isolated DB.
